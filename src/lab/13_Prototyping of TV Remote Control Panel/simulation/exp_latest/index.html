TV Remote Control

     View sample layout
There are three parts of the screen
a) Function keys (on top)
b) Movable panel elements (Buttons/labels/LED's)
c) A fixed panel cut-out (at bottom)


1. You can drag any of the movable panel elements and place them wherever you want inside the black border line of the panel cut-out.
2. A reference grid for proper placement of panel elements can be displayed using function keys on the top.
3. Once the entire prototype is ready on the panel cut-out you should freeze the movable elements by clicking FREEZE/UNFREEZE button. You can save this final prototype by using the print-screen key on the keyboard and saving it using paint or other image editing tool.
4. To exit this prototyping experiment press EXIT button on top.
Design Solution Checklist


Carefully evaluate your design. Check the item below if the statement is TRUE for your solution.

Click on "Calculate Design Efficiency" button at the end.

Displays
Display elements and labels on panel easily readable to the user
Panel designed considering vertical and horizontal position of user w.r.t. panel
Color and technology of display used matches to ambient light condition
LED positions are highlighted

Control Elements
Control elements suits the product in general
Labels on control elements are simple to understand
Each control panel element has been categorized into input and output
Type of information flowing to or from control element has been identified
Context and frequency of usage of each control element is known
Available size of panel layout is known
Human anthropometric data is known
Sizes of control elements have been designed
Relative positions of grouped control elements have been designed
Distances between control elements is designed

Ergonomics
Operational sequence of actions for each atomic task has been documented
Control elements have been placed based on operational sequence of actions
Controls elements have been grouped based on importance of tasks involved
Controls elements have been grouped based on frequency of usage in tasks
Controls elements have been grouped based on related functionality
Most frequently used control elements are easily accessible and usable
More important or used controls are placed at centre
Flow path of hand movements and eye movements for each task is analyzed
After designing the wireframe extra cognitive or motor stress is identified
Control element and task matrix computed to identify most frequently used controls
Environment/ambience of use has been identified
User's posture while interacting (seated, standing, moving) has been identified
User's viewing angle during interaction has been identified
Grouped controls and displays are clearly demarcated (bold lines/colors etc.)
Information on panel is displayed economically and is simple to understand
Visual displays are located in the upper half of the panel
Maximum buttons in a row do not exceed four
Most frequently used controls are placed on right side
Type face, size and color and position of the labels are carefully selected
CAPITAL letter forms is avoided
Color of graphics is in contrast to color of background

Aesthetics
Controls placed based on symmetry, repetition, positive & negative spaces & laws of Gestalts
Grid is used to place control elements on panel
Sufficient visual space in-between the various elements/groups is provided
Panel is visually balanced (placing or distributing controls of various shapes over entire area)
Name of instrument is carefully placed and is not be conflicting with other elements
